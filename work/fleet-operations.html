A five-car instrumented fleet run on live telemetry

I operate a five-car fleet of instrumented Tesla vehicles, with Tessie telematics streaming the live signal that drives each operational decision.

The same discipline as the agent work, against real money and real liability: real-time data in, guardrails on what acts automatically, and a human on the calls that carry the risk. This is the operator credential that makes the guardrails philosophy concrete rather than theoretical.

The problem

Running a five-car fleet is a real-time operations problem with money on every decision. Turnaround between uses, pricing, and the condition of each vehicle all move continuously, and running it on intuition leaves value and safety on the table. The need was the same one the agent work answers: instrument the system so decisions run on live data, automate the parts that are safe to automate, and keep a human firmly on anything that moves money or cannot be undone.

Approach

The fleet runs on Tessie telematics as the live signal, and turnaround and pricing decisions are made off that data rather than off guesswork. The system is decision-advisory by design: it surfaces what the data says and what it would recommend, and the human approves anything that carries money or is irreversible. It is the same capability-then-leash pattern as the agent work, applied to physical assets and real liability instead of a software action.

  • Run on the live signal. Tessie telematics streams the real-time state of each vehicle, and that signal drives turnaround and pricing decisions. The leash: data informs the call, it does not silently execute it.
  • Advise, do not transact. The instrumentation recommends the turnaround and pricing move the data supports. The leash: it stays decision-advisory, so a recommendation is not an action.
  • Keep the human on the money. Operational signals are automated; consequential moves are not. The leash: anything that moves money or is irreversible waits for the operator.

Guardrails

What the system is structurally not allowed to do. This is the through-line: capability, then leash.

  • NoIt cannot move money on its own. The human approves anything financial.
  • NoIt cannot take an irreversible action unattended. Those calls wait for the operator.
  • NoIt does not act beyond advising. The system surfaces the decision; the human owns the risk.

My role

I built and operate this fleet myself. It is my own venture, and it is where the guardrails philosophy meets real money and real liability.

Links and verification

This is a private operation, my own venture, so there is no public repo to link and no public metric to cite. It is included as the operator credential behind the guardrails philosophy: the same discipline as the agent work, applied against real money.
